# Developer Setup Guide — TDS Recon

## What Is This?

An AI-powered TDS (Tax Deducted at Source) reconciliation system. It matches Form 26 government records against Tally accounting books, flags compliance issues, and generates reports.

## Prerequisites

- **Python 3.11+** — https://www.python.org/downloads/
  - Check "Add to PATH" during install
- **Node.js 18+** — https://nodejs.org/ (LTS version)
- **Git**

## Setup (One Time)

### 1. Clone

```bash
git clone https://github.com/Scaleupcfoai/aitdsrecon.git
cd aitdsrecon
git checkout claude/sub-agents-mvp-guide-Akcin
```

### 2. Install backend dependencies

```bash
cd tds-recon
pip install -r requirements.txt
```

If `pip` doesn't work, try `python -m pip install -r requirements.txt`

### 3. Install frontend dependencies

```bash
cd ../ui
npm install
```

## Running the App

Open **two terminals**.

### Terminal 1 — Backend (API Server)

```bash
cd aitdsrecon/tds-recon
python -m uvicorn api_server:app --reload --port 8000
```

You should see:
```
INFO:     Uvicorn running on http://127.0.0.1:8000
```

### Terminal 2 — Frontend (React UI)

```bash
cd aitdsrecon/ui
npm run dev
```

You should see:
```
VITE v7.x.x  ready
➜  Local: http://localhost:5173/
```

### Open Browser

1. Go to the URL from Terminal 2 (e.g. `http://localhost:5173/`)
2. Click **"Reconciliations"** in the left sidebar
3. Click **"TDS Payable (all sections)"** tile (red TAX badge, bottom row)
4. In the chat panel, type **"run reconciliation"** or click the action chip

## How It Works

```
User clicks "Run Reconciliation"
    ↓
React UI opens SSE connection to /api/run/stream
    ↓
FastAPI runs the pipeline in a background thread:
    Parser Agent → Matcher Agent → TDS Checker → Reporter Agent
    ↓
Each agent emits events via EventLogger → SSE → Browser
    ↓
Results appear in real-time in the chat panel
Dashboard (left) updates with KPIs and section details
    ↓
User can: explore tabs, export Excel report, ask questions in chat
```

## Key API Endpoints

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/status` | Check if data exists |
| GET | `/api/run/stream` | Run full pipeline (SSE streaming) |
| POST | `/api/upload` | Upload Form 26 + Tally XLSX |
| GET | `/api/run/stream/upload` | Run pipeline on uploaded files (SSE) |
| GET | `/api/download/{filename}` | Download reports (Excel/CSV/JSON) |
| GET | `/api/results` | Get cached results |
| POST | `/api/review` | Submit human review decisions |
| POST | `/api/run/parser` | Run only Parser Agent |
| POST | `/api/run/matcher` | Run only Matcher Agent |
| POST | `/api/run/checker` | Run only TDS Checker |
| POST | `/api/run/reporter` | Run only Reporter Agent |

## Switching Clients

To clear old client data before running on a new client:

```bash
cd tds-recon
python reset_client.py --keep-rules
```

Then upload new Form 26 + Tally files via the UI.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `uvicorn not recognized` | Use `python -m uvicorn api_server:app --reload --port 8000` |
| `npm not recognized` (PowerShell) | Use Command Prompt (`cmd`) instead, or run `Set-ExecutionPolicy RemoteSigned` in admin PowerShell |
| `CORS error` / `500 error` in browser | Check Terminal 1 for the Python traceback — that's the real error |
| `Port already in use` | Kill the process: `lsof -ti:8000 | xargs kill` (Mac/Linux) or find it in Task Manager (Windows) |
| Blank page in browser | Make sure Terminal 2 (npm run dev) is running |
| `Module not found` errors | Run `pip install -r requirements.txt` again |
